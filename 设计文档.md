# 设计文档

**项目**：智能鞋垫数据采集与压力标定系统  
**版本**：v0.1.0  
**技术栈**：Python 3.11 / PySide6 6.8 / pyqtgraph 0.13 / NumPy / SciPy / PySerial

---

## 1. 整体架构

系统按职责划分为四层，层间通过 **Qt 信号/槽**解耦，数据始终单向流动。

```
┌─────────────────────────────────────────────────────────────────┐
│                         硬件设备                                  │
│             鞋垫 USB 串口  921600 baud / 8N1                      │
└─────────────────────────┬───────────────────────────────────────┘
                          │ 原始字节流
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  通信层  communication/                                           │
│                                                                   │
│  SerialManager (QThread)                                          │
│  ├─ 非阻塞 read() 循环，自动检测断线并重连                          │
│  └─ Signal: data_received(bytes)                                  │
│                              │                                    │
│  ProtocolParser (QObject)    │ 接收原始字节                        │
│  ├─ 帧头同步 [AA 55 03 99]   │                                    │
│  ├─ 双子包缓存 + 超时清理（0.5s）                                  │
│  └─ Signal: frame_merged(MergedFrame)  ← 完整帧（272B）           │
└─────────────────────────┬───────────────────────────────────────┘
                          │ MergedFrame (128B ADC + 16B IMU)
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  数据层  data/                                                    │
│                                                                   │
│  DataProcessor (QObject)                                          │
│  ├─ ADC 解码：reshape(16,8)，左右/上下翻转对齐足底方位              │
│  ├─ 噪声过滤（阈值 < 10 → 0）                                     │
│  ├─ 标定换算：P = a × (ADC − zero_offset) + b（分区线性模型）      │
│  ├─ 指标计算：总压力、CoP、峰值、FPS                               │
│  └─ Signal: frame_processed(ProcessedFrame)                       │
│                                                                   │
│  CalibrationEngine (QObject)   DataRecorder (QObject)             │
│  状态机：5 阶段标定流程          队列异步写 CSV，批量 flush          │
└─────────────────────────┬───────────────────────────────────────┘
                          │ ProcessedFrame
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  表现层  ui/                                                      │
│                                                                   │
│  MainWindow                                                       │
│  ├─ QTimer（30 FPS）：批量消费队列帧，一次性更新所有视图             │
│  ├─ HeatmapView  — RBF 插值热力图 + 软边界足底形 + 色图切换         │
│  ├─ MetricsPanel — 总压力 / CoP / 峰值 / FPS 大字号面板            │
│  ├─ TimeSeriesView — 滚动时序曲线，通道/区域/时间窗口可选            │
│  ├─ IMUView      — 四元数 + 欧拉角 + 可选 3D 姿态                  │
│  ├─ SerialPanel  — 端口选择、连接控制、统计显示                     │
│  └─ CalibrationWizard — 5 步标定向导                              │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 关键技术选型及理由

### 2.1 GUI 框架：PySide6

**选择理由**：

- Qt6 原生 C++ 渲染，控件响应与重绘性能远优于 Tkinter/wxPython 等纯 Python 方案
- `QThread` + 信号/槽机制天然支持多线程与 UI 解耦，无需手写锁或轮询
- `QWizard` 提供开箱即用的多步向导框架，标定流程与之高度契合
- 跨平台（Windows/macOS/Linux）分发成熟

**替代方案及放弃原因**：

| 备选 | 放弃原因 |
|------|---------|
| PyQt5 | 授权为 GPL，PySide6 使用 LGPL，商业更友好 |
| Electron | JavaScript 生态，与 Python 科学计算栈割裂，增加打包复杂度 |
| Tkinter | 无高性能绘图能力，200Hz 下渲染明显卡顿 |

### 2.2 可视化库：pyqtgraph

**选择理由**：

- 底层基于 Qt `QGraphicsView` + NumPy 直接渲染，`ImageItem.setImage()` 单次调用可在 ≤2ms 内完成 160×80 热力图更新
- 时序曲线使用 `PlotCurveItem`，百帧级数据点实时滚动不掉帧
- 与 PySide6 深度集成，无需跨进程通信

**与 matplotlib 对比**：matplotlib 在 200Hz 下每帧重绘耗时约 30-80ms，无法满足实时性需求；pyqtgraph 同等场景 <5ms。

### 2.3 串口线程模型：QThread + 非阻塞读取

串口读取在独立 `QThread` 中运行，使用 `serial.read(serial.in_waiting or 1)` 非阻塞模式：

- `in_waiting > 0` 时批量读取，减少系统调用次数
- `in_waiting == 0` 时读 1 字节阻塞等待，避免空转 CPU 100%
- 读取结果通过 `data_received` 信号跨线程投递到协议解析器

这样串口 I/O 完全不占用主线程，UI 不会因读串口而卡顿。

### 2.4 热力图插值：RBF（径向基函数）

原始传感器阵列为 16×8，直接显示像素化严重，采用 RBF 空间插值将传感器点值扩展到 160×80 画布：

- 传感器物理位置根据鞋垫规格书（JQGY-YL-668）映射到足底坐标系
- RBF 权重在启动时**一次性预计算**，运行中仅做矩阵乘法（O(N_sensors × N_pixels)），保证实时性
- 脚趾区使用更小的 RBF σ（0.55 vs 1.0），减少相邻趾峰融合
- 足底软边界通过 Gaussian 衰减实现，压力区域从平面"浮现"而非硬裁切

### 2.5 标定模型：分区线性拟合

**模型**：`P_kPa = a × (ADC − zero_offset) + b`（每个足底分区独立一组参数）

**设计考量**：

- 分区（后跟/足弓/前掌/脚趾）传感器受力特性不同，统一拟合误差大
- 线性模型在 0-255 ADC 范围内与薄膜压力传感器的响应曲线拟合度较高（R² 通常 > 0.98）
- 零点偏移（`zero_offset`）单独采集并逐区保存，消除初始偏压差异
- 参数以 JSON v2.0 格式持久化，支持跨鞋垫加载与版本溯源

**加分实现**：向导中同时提供二次多项式拟合选项，当线性 R² < 0.95 时自动提示。

### 2.6 数据录制：异步队列

`DataRecorder` 使用 Python `queue.Queue` + 后台 `threading.Thread`：

- 主线程将 `ProcessedFrame` 放入队列后立即返回，不阻塞采集链路
- 写线程每积累 100 帧或 0.2 秒执行一次批量 `flush`，均衡 I/O 与延迟
- CSV 按列固定宽度：`timestamp_ms, sensor_type, frame_index, ch_000~ch_127, imu_w/x/y/z, fps`

## 3. 数据流时序

```
200Hz 硬件帧率                30Hz UI 刷新率
    │                              │
    ▼                              ▼
SerialManager                 QTimer(33ms)
  │ data_received                  │
  ▼                                │
ProtocolParser                     │
  │ frame_merged                   │
  ▼                                │
DataProcessor                      │
  │ frame_processed ──────────► MainWindow._on_frame_processed()
  │                               将帧追加到 _pending_frames 队列
  │                               │
  │                               ▼ 定时器触发
  │                          取出所有 pending 帧
  │                          只用最新一帧更新 UI
  │                          丢弃过时帧（降频渲染）
  │
  └──────────────────────────► DataRecorder（所有帧，不丢帧录制）
```

> UI 以 30 FPS 消费最新帧，但 DataRecorder 接收所有 200Hz 帧，保证录制完整性。

## 4. 异常恢复策略

| 故障场景 | 检测方式 | 恢复措施 |
|---------|---------|---------|
| 串口意外断开 | `serial.read()` 抛 `SerialException` / `OSError` | 关闭句柄，状态置 `ERROR`，下一帧循环重新尝试打开；`reconnect_count` 计数 |
| 协议帧头丢失 | `_buffer[:4] != FRAME_HEADER` | 向后搜索下一个合法帧头，丢弃前置乱码 |
| 双包合并超时 | `PACK1` 或 `PACK2` 等待超过 0.5s | 丢弃孤包，`frames_drop` 计数+1 |
| 非法序列号 | SEQ 不为 0x01/0x02 | 记录 `format_err`，继续处理后续数据 |
| 主线程未捕获异常 | `sys.excepthook` | 写入 `logs/app.log`，弹出 `QMessageBox` 后优雅退出 |
| 子线程未捕获异常 | `threading.excepthook`（Python 3.8+） | 写入日志，主窗口状态栏显示错误提示 |

## 5. 性能保障措施

| 环节 | 优化措施 |
|------|---------|
| RBF 插值 | 启动时预计算权重矩阵，运行时只做 `weights @ adc_values`（NumPy 矩阵乘法） |
| UI 刷新 | 30 FPS 定时器限帧，200Hz 数据流只取最新一帧渲染，防止积压 |
| CSV 写盘 | 后台线程批量写入（100帧/批），不阻塞主采集链路 |
| 时序曲线 | `collections.deque(maxlen=N)` 滚动窗口，自动丢弃超出时间窗口的历史数据 |
| 协议解析 | `bytearray` 原地操作，避免频繁内存分配；孤包超时清理防止无限累积 |

压力测试（`tests/stress_test.py`）验证：200Hz 连续输入 5 分钟，内存稳定，无帧积压。

## 6. 标定流程详解

```
步骤 0：准备
  └─ 输入砝码接触面积（cm²）
  └─ 空载采集 1-2s → 自动生成有效传感器掩码（mask）
  └─ 可视化掩码，支持手动调整分区行边界

步骤 1：零点校准
  └─ 鞋垫空载静置 3s
  └─ 逐区计算 zero_offset（区内有效 ADC 均值）

步骤 2：多点采集
  └─ 用户依次放置砝码（建议：0/1/2/5/10/20 kg）
  └─ 每个重量点：弹出非阻塞输入框输入力值(N)，采集 3s
  └─ 每区独立记录 CalibrationPoint（avg_adc, std, sample_count）
  └─ 支持多位置（前/中/后）重复采集提高覆盖率

步骤 3：曲线拟合
  └─ 每区独立线性回归：P = a*(ADC - offset) + b
  └─ 可选二次多项式拟合
  └─ 可视化散点 + 拟合曲线 + R² 系数

步骤 4：保存与应用
  └─ 导出 JSON（CalibrationProfile v2.0）
  └─ 同时导出原始采集帧 CSV（供事后分析）
  └─ 可立即应用到主数据流（DataProcessor 载入参数）
```

## 7. 已知限制与改进思路

### 7.1 当前限制

| 限制 | 描述 |
|------|------|
| 单脚支持 | 当前只处理右脚鞋垫；左脚需镜像翻转逻辑尚未接入 |
| 线性标定模型 | 薄膜传感器在高压段存在非线性，线性拟合在极端载荷下误差偏大 |
| 标定范围依赖操作者 | 砝码放置位置不均匀时，分区采集数据代表性不足 |
| 无数据回放 | 当前不支持加载历史 CSV 文件进行回放（仅录制） |
| CoP 轨迹无持久化 | 步态分析中 CoP 轨迹仅实时显示，不存储 |
| 端口热插拔 | 依赖手动点击「刷新」，未实现 OS 级 USB 热插拔事件监听 |

### 7.2 改进方向

**短期（功能完善）**

- **数据回放**：加载历史 CSV，以原始帧率重放，用于离线分析与演示
- **CoP 轨迹绘制**：在足底热力图上叠加滚动 CoP 轨迹，辅助步态分析
- **左右脚对比**：支持同时连接两个鞋垫，双图并排显示，实时对比左右足压力分布
- **端口热插拔**：监听 `WM_DEVICECHANGE`（Windows）或 `udev` 事件，自动刷新端口列表

**中期（精度与可靠性）**

- **非线性标定**：引入分段线性或二次样条拟合，改善高压段精度
- **温漂补偿**：采集环境温度并加入标定参数，减少温度引起的零点漂移
- **滑动窗口零点更新**：利用无负载帧自动修正零点偏移，无需每次重新标定

**长期（产品化）**

- **步态周期自动检测**：基于总压力曲线的峰值检测，自动切割步态周期
- **多鞋垫型号支持**：将传感器布局（行列数、物理尺寸、分区划分）提取为外部 JSON 配置
- **云端数据上传**：支持 MQTT/WebSocket 将采集帧流式上传到云端数据库
- **打包分发**：使用 `PyInstaller` 或 `Nuitka` 生成单文件 `.exe`，降低部署门槛
