# Git 提交规范

## 提交信息格式

建议使用：

```text
<type>: <简短主题>
```

常用 `type`：

- `feat`：新功能
- `fix`：缺陷修复
- `refactor`：重构（不改行为）
- `test`：测试相关
- `docs`：文档更新
- `chore`：构建或工具链维护

示例：

- `feat: 增加分区标定结果即时应用`
- `fix: 修复串口断开后状态未更新问题`
- `docs: 补充运行说明与故障排查`

## 提交粒度

- 一次提交只做一件事
- 不混入无关格式化改动
- 保持可回滚、可定位

## 提交前检查

- 本地可启动：`python main.py`
- 关键测试通过：`pytest`
- 无明显日志异常（`logs/app.log`）

## 分支建议

- `main`：稳定分支
- `feature/*`：功能开发
- `fix/*`：问题修复
- `docs/*`：文档更新
