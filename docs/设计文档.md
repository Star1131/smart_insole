# 设计文档

## 1. 目标

本系统用于智能鞋垫实时数据采集、解析、可视化与压力标定，核心目标：

- 在 200Hz 数据输入下稳定运行
- 串口采集与 UI 渲染解耦，避免互相阻塞
- 提供可重复的标定流程与参数落盘能力

## 2. 架构概览

- `communication`：串口与协议解析
- `data`：数据解码、指标计算、标定与录制
- `ui`：主窗口与可视化组件
- `utils`：数学工具

数据流：

1. `SerialManager` 读取原始字节流
2. `ProtocolParser` 完成帧同步与双包合并
3. `DataProcessor` 计算指标与分区结果
4. UI 以定时器限帧方式刷新最新结果

## 3. 异常恢复设计

- 全局异常：`main.py` 注册 `sys.excepthook` 与 `threading.excepthook`
- 串口异常：`SerialManager` 捕获 `SerialException/OSError`，关闭句柄后重试
- 协议异常：非法序号与超时孤包丢弃并统计

## 4. UI 设计要点

- 深色主题统一风格（按钮/输入框/菜单/进度条）
- 关键指标大字号显示，降低现场观察成本
- 热力图支持配色切换与分区标注

## 5. 标定流程

1. 空载零点采集
2. 多砝码点采集
3. 分区线性拟合（`P = a*(ADC-offset)+b`）
4. 导出 JSON 并可立即应用

## 6. 测试建议

- 单元测试：协议解析、标定拟合、数学函数
- 压测：200Hz 连续输入 5 分钟以上
- 故障注入：随机噪声、截断包、串口拔插
